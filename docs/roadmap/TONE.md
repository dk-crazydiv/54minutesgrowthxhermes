# Tone

Write the way you talk. Simple words, short sentences. If you wouldn't say it out loud
to a teammate, don't write it.

- Everyday words. Not "utilize", "leverage", "facilitate". Use, help, do.
- Periods and commas. If a sentence needs a semicolon, it's two sentences.
- Real specifics. The real file name, the real number, the real error line.
  "The gateway died when the shell closed" beats "stability concerns exist."
- Delete what's done. Stale finished items make nobody trust the doc. Git remembers.
- No report voice, no drama, no nicknames for incidents. Plain cause and effect that
  someone who wasn't there can follow.
- Every status update leads with what's broken, what's in flight, and what's waiting
  on Kartik. Never say "all routine" while a known problem is open.
- Read it out loud once before you commit. If you trip, fix it.
